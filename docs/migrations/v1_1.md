# V1.1 Database Migration

V1.1 adds one new Paper field and tightens Tag color behavior.

## Schema Changes

- `papers.last_read_at TEXT NULL`
- `tags.color` should no longer be treated as optional by the application.
- Existing tags with `NULL` or empty colors are backfilled to `#E2E8F0`.
- New index: `idx_papers_last_read_at ON papers(last_read_at DESC)`

## Apply Migration

Stop the backend server first, then run from the repository root:

```bash
.venv/bin/python scripts/migrate_v1_1.py
```

Expected output:

```text
Added papers.last_read_at
V1.1 migration completed
```

If the migration has already been applied, the script is idempotent:

```text
papers.last_read_at already exists
V1.1 migration completed
```

## Fresh Database

If you delete `data/paper_reader.db` and restart the FastAPI app, `SQLModel.create_all()`
will create the latest table shape automatically. The migration script is only needed
for an existing SQLite database.

## Verify

After migration, start the backend and check:

```bash
.venv/bin/python -m uvicorn app.main:app --reload
```

Then visit:

```text
http://127.0.0.1:8000/docs
```

The `PaperRead` schema should include `last_read_at`, and `GET /api/papers` should
accept `sort_by=year`, `sort_by=year_desc`, `sort_by=last_read_at`, and
`sort_by=last_read_desc`.
